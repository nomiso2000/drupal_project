Типове створення та викорситаненя кастомного плагіна.
Для створення самого плагіна, використано drush gen plugin:manager
Для створення форми, яка керуватиме роботою плагіна викоистано drush gen form:config

Для пошуку та створення плагінів в даному випадку використано Анотацію.
Це дозволяє створювати під-плагіни, які будуть знаходитися та визначатися PluginManager - ом
PluginInterface - визначить які методи будуть доступні в нашому PluginBase.
Нашіж плагіни в свою чергу будуть наслідувати PluginBase.
Для створення кастомної поведінки, було створено .module, для впровадження хуків.
Для того що б зручно можна було при виконанні певного хука, викликати наші плагіни,
було створено Service та .services.yml, це дозволить легко та зручно викликати виконання плагіна з заданими параметрами.

Форма
В формі ми виводимо існуючі під-плагіни, і за допомогою .schema створюємо конфігураційний файл.
(Схема потрібна для створення відповідної структури конфіг. файла, та і в цілому для його створення)
Це дозволить в самому сервісі діставати конфіг файл, (не забувай що треба передати  @config.factory в .services)
